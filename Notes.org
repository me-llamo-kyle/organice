* Monads son
** AI: ELI5 what is a monad?
Think of a monad as a special safety wrapper around a regular value. It is like a secure shipping box that protects whatever is inside while letting you safely process it.
*** The Problem it Solves
Imagine you have a regular function that takes a number, cuts it in half, and spits out the result.
If you pass it the number 10, it gives you 5.
But what happens if you pass it nothing (empty space)? The code will crash.
*** How a Monad Helps
Instead of leaving your values naked, you put them inside a monad wrapper. The wrapper has three superpowers:
1. 🔒 The Wrapper: It holds the value safely. It might say, "Inside here is the number 10," or it might say, "This box is completely empty."
2. ⚙️ The Conveyor Belt: You can send a regular instruction sheet (a function) into the box. The box automatically opens up, applies your instruction to the value inside, and puts the new result into a brand new box.
3. 💥 Safety Early-Exit: If the box is empty, it skips all your instructions entirely. It won't crash; it just passes the empty box down the line.
** AI: How is that a monoid in the category of endofunctors?
A monad is a monoid in the category of endofunctors because it fulfills the exact definition of a monoid, but using types and functions instead of numbers.
To understand this famous quote, we can break it down by comparing a normal school-math monoid to a category-theory monoid.
*** What is a Monoid? (Simple Math)
In basic math, a monoid is just a set of things, a way to combine them, and a "do-nothing" element.
For example, Numbers with Addition:
A set of things: Numbers (\(1, 2, 3...\)).
A combination rule: Addition (\(+\)). If you combine two numbers, you get another number (\(2 + 3 = 5\)).
An identity element: Zero (\(0\)). If you add it to anything, nothing changes (\(5 + 0 = 5\)).
*** What is an Endofunctor? (The Wrapper)
In category theory, an endofunctor is a map that takes types and functions from one category and maps them back into that same category. In programming, you can just think of it as a generic type wrapper with a .map() function.
Option<T> is an endofunctor.
It takes a type T (like i32) and maps it to a new type Option<i32>.
It takes a function f: T -> U and maps it to a function Option<T> -> Option<U> using .map().
*** Putting it Together: A Monoid of Wrappers
A monad is a monoid where the "things" being combined are these wrapper layers (endofunctors).
To be a monoid, it needs a combination rule and a do-nothing element:
The Do-Nothing Element (Identity / Pure)
Instead of adding \(0\), a monad has a way to turn a normal value into a wrapped value.
In category theory, this is a natural transformation called \(\eta \) (eta).
In Rust, this is Some(x) or Ok(x).
It introduces the context without changing the data.
The Combination Rule (Multiplication / Join / Flatten)
Instead of adding numbers (\(2 + 3\)), a monad needs a way to combine nested wrappers into a single wrapper (\(M(M(T)) \rightarrow M(T)\)).
In category theory, this is a natural transformation called \(\mu \) (mu).
In programming, if you map a function that returns a monad, you get a nested monad: Option<Option<T>>.
The combination rule flattens it back down to a single Option<T>. In Rust, this is .flatten().
** NI
A functor in programming can basically be thought of as a wrapper, one that lets you both wrap objects /and/ "wrap" functions (by using .map()).

This describes an endofunctor on the category of types: it maps type T to type Result<T>, and maps functions f: S -> T to .map(f): Result<S> -> Result<T>.

Example of a functor that isn't an endofunctor - transpiler that maps types from one language onto types in another language, and likewise with functions.

What's special about monads is the multiplication/flattening map (also that you can always wrap a value). That allows you to for instance map a function through a Result<T>, ordinarily giving a Result<Result<T>>, and flatten that to a single Result<T>. That way you can keep a potentially bad result wrapped, try to do some stuff to it, and only have one layer of wrapping at the end to unwrap.

.unwrap() isn't related to being a functor/monad - mathematically it's part of a comonad.
* Comonads daughter
A comonad has an extract method (like unwrap but never panics) and a duplicate method
e :: w t -> t
d :: w t -> w (w t)
Allows you to add context and perform a context-dependent computation
Example: Conway's Game of Life - each cell updated based on context of its neighbors
You'd use duplicate to replace the grid with a grid of 3x3 local neighborhoods, then use fmap (functorial) to replace each neighborhood with the corresponding new square
